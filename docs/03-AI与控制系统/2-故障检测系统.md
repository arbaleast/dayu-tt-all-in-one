# 故障检测系统

> 大鱼 TT 的打印安全网：多层次 AI 故障检测，从"事后发现"到"实时拦截"

---

## 1. 系统概述

3D 打印过程中，常见的故障类型包括：

| 故障类型 | 描述 | 检测方式 |
|---------|------|---------|
| **Spaghetti**（意大利面）| 耗材从喷嘴脱落并在平台外缠绕 | 视觉（摄像头）|
| **Blob / Zhop** | 喷嘴撞击打印件，熔融耗材堆积 | 视觉 + 振动 |
| **Warp / 翘曲** | 热床温度不足导致首层剥离 | 视觉（首层图像）|
| **Layer Shift**（层移）| 某层开始后 XY 位置突然偏移 | 视觉 + 振动 |
| **喷嘴堵塞** | 耗材无法正常挤出，流量骤降 | 视觉 + 挤出机负载 |
| **热床失衡** | 热床加热不均匀导致局部翘曲 | 热敏电阻数据 |

大鱼 TT 的故障检测系统采用**多传感器融合**策略，以视觉检测为主，振动检测为辅，实现故障的早发现、早干预，避免因小故障导致长时间打印报废。

## 2. Obico AI（已集成 Phase 0）

### 简介

**Obico**（原名 The Spaghetti Detective）是 3D 打印领域最成熟的 AI 故障检测平台，通过深度学习模型实时分析打印画面，一旦检测到 Spaghetti 或其他异常，立即通过 WebUI / App / Telegram 推送告警，并可自动暂停打印。

Obico **已于 Phase 0 集成到大鱼 TT 项目中**，开箱即用。

### 核心能力

| 指标 | 数据 |
|------|------|
| 检测类型 | Spaghetti、Blob、Layer Shift、喷嘴堵塞、翘曲 |
| 免费额度 | 每月 10 AI 小时（云端推理）|
| 付费方案 | $5/月（无限 AI 小时）|
| 部署方式 | 云端 SaaS / 本地 Docker |
| 推送渠道 | WebUI、Moonraker App、Telegram、Email |
| 识别准确率 | Spaghetti 检出率 > 95%（官方数据）|

### 架构原理

```
摄像头 → Moonraker（Klipper API）→ Obico Agent（AI 推理）→ 告警/暂停
```

摄像头持续拍摄打印画面，Obico Agent 逐帧分析图像，当检测置信度超过阈值（如 80%）时触发告警。

### 扩展配置

**进阶选项：提高检测灵敏度**

```yaml
# moonraker.conf 中 Obico 插件配置
[octoprint_compat]
obico:
  detection_sensitivity: high  # 可选: low / medium / high
  auto_pause: true
  notification_channels:
    - telegram
    - app
  spagghetti_confidence_threshold: 0.75  # 置信度阈值
```

**本地化部署（无隐私顾虑）**

Obico 支持完全本地化部署，使用 Docker 在本地运行推理模型：

```bash
# 使用 Docker 部署 Obico 本地服务器
docker run -d \
  --name obico-server \
  -p 3333:3333 \
  -v ~/obico_data:/data \
  theSpaghettiDetective/obico-server:latest
```

本地部署后，所有图像数据不离开局域网，隐私安全无忧。

### 已知局限

- 免费版每月仅 10 AI 小时，连续长时间打印可能耗尽配额
- 对某些非 Spaghetti 的复杂缺陷（如轻微 stringing）识别较弱
- 依赖稳定的网络连接（云端模式）

## 3. OctoEverywhere 免费无限量方案

### 简介

**OctoEverywhere** 是一个完全免费的 3D 打印监控平台，核心优势是**无限量免费使用**，不限制 AI 推理时长。与 OctoPrint 或 Mainsail/Fluidd 配合使用。

### 核心能力

| 指标 | 数据 |
|------|------|
| AI 检测 | 无限量（完全免费）|
| 检测类型 | Spaghetti、Blob、层移、挤出异常 |
| 机型支持 | Bambu Lab、Prusa、Creality、Voron + 通用 OctoPrint |
| 远程访问 | 免费内网穿透，无需公网 IP |
| App | iOS / Android 原生 App |

### 与 Klipper 的集成

OctoEverywhere 通过 OctoEverywhere Gadget 与 Moonraker 通信：

```bash
# 安装 OctoEverywhere 插件
cd ~/klipper && make menuconfig
# 启用 OctoEverywhere 支持后重新编译
```

或通过 Mainsail/Fluidd 的 Extension Manager 直接安装 `OctoEverywhere` 插件。

### 优势 vs Obico

| 对比项 | OctoEverywhere | Obico |
|-------|--------------|-------|
| 免费额度 | **无限量** | 每月 10 小时 |
| 检测精度 | 良好（CNN 模型）| 优秀（专用模型）|
| 本地部署 | 支持 | 支持 |
| 移动 App | ✅ | ✅ |
| 告警渠道 | App + 邮件 | App + Telegram + 邮件 |
| 远程访问 | 免费内网穿透 | 需要配置 |

> 💡 **推荐策略**：如果打印量大、预算有限，优先使用 OctoEverywhere 作为主力监控；Obico 作为备用或复杂故障的二次确认。

## 4. 3DPrintSentinel 完全本地化方案

### 简介

**3DPrintSentinel** 是追求数据隐私用户的最佳选择——模型完全在本地运行，图像数据不上传任何服务器。采用与 Obico 相似的深度学习模型架构，专门针对 Spaghetti 检测优化。

### 核心特点

| 特点 | 说明 |
|------|------|
| 完全本地化 | 100% 离线运行，无网络需求 |
| 模型复用 | 基于 Obico 社区开源模型 fine-tune |
| 树莓派原生 | 专为 ARM 架构优化，无需 GPU |
| 隐私优先 | 图像数据不离开设备 |
| 目标用户 | 企业/重视隐私的个人用户 |

### 部署方式

```bash
# 在树莓派上安装 3DPrintSentinel
git clone https://github.com/Red-Beard/3DPrintSentinel.git
cd 3DPrintSentinel
./install.sh

# 启动服务
3dprintsentinel start
```

### 局限

- 纯视觉检测，不支持振动融合
- 模型更新需要手动拉取新版本
- 不支持远程访问（纯本地）

## 5. 双传感器融合：视觉（Obico）+ 振动（MPU6050）

### 为什么需要传感器融合？

单一视觉检测存在盲区：
- **摄像头角度不佳**：喷嘴下方或角落的异常难以发现
- **反光/蒸汽干扰**：某些材料打印时水汽遮挡画面
- **刚性问题**：某些故障（如轻微层移）在视觉上不够明显

振动检测可以捕捉**物理层面的异常**，是对视觉检测的有力补充。

### MPU6050 振动检测方案

MPU6050 是一款六轴 IMU（3 轴加速度 + 3 轴陀螺仪），成本约 ¥5，可挂在 Klipper 的 I2C 总线上：

```ini
[mcu dummy]
# 假设 MPU6050 连接在树莓派 I2C 总线上
[mpu6050]
i2c_address: 0x68
i2c_mcu: klipper_mcu

[respond]
# 振动异常阈值监控宏
```

### 融合检测逻辑

```
正常状态：
  视觉检测：无异常 ✅
  振动检测：频率平稳 ✅
  → 继续打印

Spaghetti 状态：
  视觉检测：检测到异常形状 ❌
  振动检测：检测到不规则振动峰值 ❌
  → 双重确认后立即暂停

轻微层移（视觉不确定）：
  视觉检测：置信度 55%（不确定）
  振动检测：检测到明显冲击 ❌
  → 综合判断：触发警告，建议人工确认
```

### 推荐实践

| 场景 | 推荐方案 |
|------|---------|
| 日常打印（¥0 预算）| OctoEverywhere（免费无限）+ MPU6050 振动辅助 |
| 隐私敏感环境 | 3DPrintSentinel + MPU6050 |
| 高可靠性需求 | Obico（付费版）+ OctoEverywhere 双保险 |
| 最佳性价比 | OctoEverywhere + 自建 MPU6050 监控 |

### 大鱼 TT 推荐配置

大鱼 TT 作为高性能 Voron 改型，建议采用以下监控方案：

1. **主力监控**：OctoEverywhere（免费无限量）
2. **备用/高精度**：Obico（Phase 0 已集成，可按需启用）
3. **振动辅助**：MPU6050 挂在 CAN 总线上，通过 SB2040 收集数据传给 Klipper
4. **本地备份**：3DPrintSentinel 作为内网隐私兜底方案

## 6. 快速启用指南

### Obico（Phase 0 已集成）

```bash
# 在 Mainsail/Fluidd 的 WebUI 中：
# Machine → Extension → OctoEverywhere / Obico → 安装并绑定账号
```

### OctoEverywhere

```bash
# 通过 Mainsail Extension Manager 安装
# Settings → Extension Manager → 搜索 "OctoEverywhere" → 安装
# 重启 Klipper 后自动出现 OctoEverywhere Tab
```

### MPU6050 振动监控（手动配置）

```ini
# 在 printer.cfg 添加
[mpu6050]
i2c_bus: i2c1
i2c_address: 0x68

[resonance_monitor]
accel_chip: mpu6050
vibration_threshold: 15.0  # 加速度阈值（m/s²），超过则告警
```
