> 从"被动工具"到"主动进化者"——大鱼 TT 的五阶段 AI 自我演化路径

---

## 1. 概述：什么是自我演化控制？

传统自动化系统遵循固定规则：IF 故障 THEN 暂停，IF 温度 > 230°C THEN 报警。规则是工程师预先编写的，不会从历史中学习。

**自我演化控制系统** 则具备：

- **经验积累**：每次打印后提取知识，改进下次打印
- **动态优化**：根据历史数据自动调整参数，而非依赖人工经验
- **闭环反馈**：缺陷 → 根因分析 → 参数修正 → 验证 → 知识沉淀

这是从"使用 AI 工具"到"让 AI 成为打印机的一部分"的质变。

### Phase 2：串联工作流（已具备条件）

**状态**：🔧 正在集成

将 Phase 1 的单点工具通过 Moonraker API 串联成自动工作流：

```
打印前 ──▶ Klippain 一键校准（IS + 床网 + PA + Flow）
   │
   ▼
打印中 ──▶ OctoEverywhere / Obico 实时监控
   │         │
   │         ▼ 检测到故障
   │         自动暂停 + 推送告警
   │
   ▼
打印后 ──▶ 打印记录自动归档
            参数 + 质量结果写入知识库
            VLM 质检评分入库
```

**Moonraker 脚本自动化示例**：

```python
# 打印任务自动化脚本（伪代码）
class DayuTTWorkflow:
    def on_print_start(self, print_id):
        # 1. 检查距离上次校准是否 > 7 天
        if self.needs_recalibration():
            print("🔧 执行 Klippain 校准...")
            self.run_klippain_calibrate()  # IS + 床网 + PA + Flow
        
        # 2. 启动监控
        self.start_octoeverywhere()
        self.start_vlm_monitor()
    
    def on_layer_complete(self, layer_num):
        # 3. VLM 逐层质检
        quality = self.vlm_analyze_current_layer()
        if quality < 6.0:
            self.pause_and_alert("质量问题严重，已暂停")
    
    def on_print_complete(self, print_id):
        # 4. 归档打印记录
        self.archive_print_result(
            print_id=print_id,
            quality_score=self.avg_quality,
            params_used=self.current_params,
            was_successful=self.final_status == "complete"
        )
```

**状态**：⚠️ 需重点建设

```
│                 │                                          │
│                 ▼                                          │
│  [打印历史] ──▶ [Monitor Agent] ──▶ [视觉/振动输入]        │
│                 │                                          │
│                 ▼                                          │
│           [Executor Agent] ──▶ Klipper 参数动态调整         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**核心能力提升**：

| 能力 | Phase 2 | Phase 3 |
|------|---------|---------|
| 故障识别 | 二分类（正常/异常）| 多分类 + 根因推理 |
| 参数调整 | 手动修改 | 自动调整 + 验证 |
| 历史利用 | 简单归档 | 语义检索 + 相似任务推荐 |
| 知识积累 | 无 | RAG 知识库持续沉淀 |

**RAG 驱动的问题解决**：

当 Reasoner Agent 遇到新问题时，不再仅依赖规则，而是：

1. 从知识库检索相似历史案例
2. 结合 Klipper 领域知识进行推理
3. 生成可解释的诊断报告
4. 推荐具体修复步骤

**代表成果**：[RAG 知识库架构](./4-RAG知识库架构.md)、[VLM 打印质量监控](./3-VLM打印质量监控.md)

#### 在线学习策略

**不依赖大量离线数据**，通过在线探索-利用平衡实现持续优化：

```python
class AdaptiveOptimizer:
    def __init__(self):
        self.exploration_rate = 0.3  # 初始探索率 30%
        self.min_exploration = 0.05  # 最低探索率 5%
        self.learning_rate = 0.01
    
    def suggest_params(self, context):
        # Epsilon-greedy 策略
        if random.random() < self.exploration_rate:
            return self.explore_random()      # 探索：随机尝试
        else:
            return self.exploit_best()        # 利用：选择历史最优
    
    def on_print_complete(self, quality_score, params):
        # 贝叶斯更新参数-质量映射
        self.update_param_model(params, quality_score)
        
        # 逐渐降低探索率
        self.exploration_rate = max(
            self.min_exploration,
            self.exploration_rate * (1 - self.learning_rate)
        )
```

#### 仿真器加速学习

真实打印机无法无限探索（时间/耗材成本高）。使用 Klipper 仿真器进行大量虚拟打印：

```bash
# Klipper 内置仿真模式
[Klipper]
simulation_mode: true
```

在仿真器中完成参数空间探索，再将验证后的最优参数应用到真实机器。

### Phase 5：完全自主的 3D 打印 Agent（长期愿景）

**愿景**：用户只需说"帮我打印这个模型，最优质量"，系统自主完成从校准到完成的全过程，无需任何人工干预。

```
用户输入：
"用大鱼 TT 打印这个文件，目标是最高表面质量，可以接受较长时间"

系统自主完成：
1. 识别文件：STL → 分析几何特征（小拐角、高速区域）
2. 选择耗材：eSUN PLA+（知识库匹配）
3. 自动校准：调用 Klippain（IS + 床网 + PA + Flow）
4. 生成切片参数：基于历史最优记录，生成切片配置
5. 启动打印：Monitor Agent 逐层监控
6. 故障处理：自动识别并修复（无需用户）
7. 质量验收：VLM 评估成品质量，记录归档
8. 知识更新：将本次经验写入知识库
```

**技术挑战**：

| 挑战 | 当前状态 | 解决路径 |
|------|---------|---------|
| 长期任务规划 | 无成熟方案 | LLM 作为 Agent 控制器 |
| 安全保障 | 需要物理级安全网 | 硬限位 + 双重监控 |
| 可解释性 | 黑盒推理 | RAG 提供因果链说明 |
| 端到端优化 | 各模块独立 | 联合强化学习 |
